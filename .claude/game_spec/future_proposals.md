# EmojiParadigm 今後の実装提案

> 作成日: 2026-05-25  
> 現状: キャラ30体 / ステージ8 / NG+対応 / ロール解放・装備強化・オプティマシステム実装済み

---

## 優先度: 高（プレイ体験に直結）

### 1. バトル速度設定
**概要**: バトル中に ×1 / ×2 / ×3 の速度倍率を切り替えられるボタンを追加  
**理由**: 周回プレイで同じバトルを繰り返す際のストレス軽減。ATBゲームの定番機能。  
**実装ポイント**:
- `atb.ts` の `ATB_TICK_MS` をグローバル倍率で割る
- `localStorage` に設定を保存する
- バトル画面右上にトグルボタン配置

---

### 2. チェーンボーナス段階強化
**概要**: チェーンゲージが特定値（200 / 500 / 999）を超えたとき、ダメージ倍率に段階ボーナスを追加  
**理由**: チェーンを伸ばすことへの動機付けが薄い。高チェーンでのカタルシスを強化する。  
**実装ポイント**:
- `combat.ts` のダメージ計算に `chainTierBonus` を追加
- チェーンゲージの色を段階ごとに変化させる（黄→オレンジ→赤→紫）

---

### 3. 装備セット（プリセット）保存
**概要**: キャラごとに「セット1〜3」として装備構成を保存・呼び出しできる機能  
**理由**: 現状は手動で付け替えが必要。複数ビルドを試したいユーザーに有用。  
**実装ポイント**:
- `CharacterSaveData` に `equipPresets: EquipmentSlots[]` フィールドを追加
- `EnhanceScreen` の装備タブにプリセット保存・適用ボタンを追加

---

### 4. ステージ9〜12（NG+専用）
**概要**: NG+1以上でのみ出現する上級ステージを4つ追加  
**理由**: NG+になるとステージ1〜8が容易すぎる。周回の目標が欲しい。  
**実装ポイント**:
- `STAGE_WAVES` に追加エントリを追加するだけで対応可能
- 既存の強敵（`finalboss`, `chaosgod` など）を複数配置した高難易度ウェーブ構成

---

### 5. バトルログ改善 / ダメージ統計
**概要**: バトル終了後のリザルト画面に「最大ダメージ / 総ダメージ / 最高チェーン / バトルタイム」を表示  
**理由**: リザルトが単調。プレイヤーが自分の強さを実感できる数値を見せる。  
**実装ポイント**:
- `BattleScreen` で統計値を `useState` で追跡し、`onVictory` に渡す
- `ResultScreen` の下部に統計パネルを追加

---

## 優先度: 中（コンテンツ・深み追加）

### 6. スキルボードの充実
**概要**: `skillBoard.ts` の基盤を活かし、キャラごとのノード解放を本格実装  
**現状**: 型定義・データは存在するが UI がほぼ未使用  
**実装ポイント**:
- `EnhanceScreen` にスキルボードタブを追加
- ノードを SVG または CSS Grid でビジュアル表示
- 解放条件: ロールLv / クリスタル消費

---

### 7. エンドレスモード
**概要**: ステージ選択から「エンドレス」を選ぶと無限ウェーブが続くモードを追加  
**理由**: NG+後の腕試し・ランキング的な楽しみ方。  
**実装ポイント**:
- 既存ステージのウェーブをランダムに繰り返し、NG+倍率をターンごとに上昇
- `ResultScreen` に「到達ウェーブ数」を表示
- `localStorage` にハイスコアを保存

---

### 8. 装備合成（クラフト）
**概要**: 低レア装備 + 素材 → 新装備を作成できるクラフトシステム  
**理由**: 素材の使い道がロールLvとロール解放のみで余りやすい。  
**実装ポイント**:
- `items.ts` にレシピデータ `CraftRecipe[]` を追加
- `ShopScreen` にクラフトタブを追加

---

### 9. キャラクタープロフィール / フレーバーテキスト
**概要**: キャラ選択画面 / エンハンス画面にキャラの説明文・得意戦法を表示  
**理由**: キャラへの愛着がゲームの継続動機になる。  
**実装ポイント**:
- `CharacterData` に `description: string` と `playstyle: string` フィールドを追加
- `AbilityViewer` か新規 `CharProfilePanel` コンポーネントで表示

---

### 10. ボスラッシュモード
**概要**: 各ステージのボス（最終ウェーブ）だけを連続攻略するモード  
**理由**: 強いビルドでボスを倒し続ける爽快感。強化ループの報酬になる。  
**実装ポイント**:
- `STAGE_WAVES` から各ステージの最終ウェーブだけを抽出してシーケンス生成
- クリアごとに Git + レア素材をドロップ

---

## 優先度: 低（UX 改善・アクセシビリティ）

### 11. セーブデータ エクスポート / インポート（URL）
**概要**: セーブデータを Base64 エンコードして URL に埋め込み、共有・バックアップ可能にする  
**理由**: `localStorage` はブラウザ依存。デバイス間移行やシェアがしたいユーザー向け。  
**実装ポイント**:
- `save.ts` に `exportToBase64() / importFromBase64()` を追加
- `SaveLoadPanel` にコピー・貼り付けボタンを追加

---

### 12. サウンドエフェクト（Web Audio API）
**概要**: 攻撃・回復・ブレイク・オプティマ切替に簡易 SE を追加  
**理由**: 視覚＋聴覚でバトルの手触りが大幅改善。  
**実装ポイント**:
- `AudioContext` + `OscillatorNode` でシンプルな矩形波 SE を生成（外部ファイル不要）
- ミュートボタンと音量スライダーを設定として追加

---

### 13. モバイルレイアウト最適化
**概要**: スマホ縦持ち時のバトル画面・強化画面のレイアウトを整える  
**理由**: GitHub Pages でのプレイはスマホユーザーが一定数いる。  
**実装ポイント**:
- `@media (max-width: 480px)` でカード横並びを縦スタックに変更
- バトル中のボタンサイズを指タップ向けに拡大

---

### 14. パーティカード画像ダウンロード
**概要**: パーティカードモーダルを PNG として保存できるボタンを追加  
**理由**: 「スクリーンショットで保存」の案内だけでは UX が粗い。  
**実装ポイント**:
- `html2canvas` ライブラリを使用して DOM → Canvas → ダウンロード
- または `OffscreenCanvas` + 手動描画で依存なしに実装
